Trading activity is not confined to desktop sessions. Market-moving events occur at all hours. The interval between signal and action is often measured in seconds. A desktop-only model has a structural blind spot: it stops learning the moment the user closes the laptop.

Omnera's mobile application extends the behavioral model beyond desktop sessions.

## How It Works

- The mobile experience is built around the same behavioral model as the desktop interface
- The interaction pattern is scroll, evaluate, execute. All three actions occur within a single screen
- The interface is optimized for single-hand use. Trade execution requires one tap
- Pre-trade intelligence surfaces inline at the moment of decision
- Every mobile interaction adds signal that refines the model regardless of whether a trade executes

## Notifications

The mobile application extends the feed beyond active sessions.

- When the behavioral model identifies a high-relevance opportunity, a notification is delivered
- Notifications are generated by the same model that ranks the feed. They are not generic price alerts
- Notification frequency is determined by the model's assessment of relevance to the individual user, not by market volatility or trending activity

## Why Mobile Matters for the Model

A user who only interacts on desktop provides behavioral signal during desktop sessions. A user who also interacts on mobile provides signal continuously. The model for a mobile-active user resolves finer behavioral distinctions because it observes decisions across a wider range of contexts, times, and market conditions.
