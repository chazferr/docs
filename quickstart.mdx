Omnera is in early access. The system is capacity-limited by design. Every user's behavioral model requires dedicated compute, and model quality depends on controlled onboarding.

## Waitlist

Access begins at omnera.io. Invitations are distributed on a rolling basis as infrastructure capacity expands. Waitlist position and community engagement both factor into selection.

## Wallet Connection

On receiving an invitation, users connect a non-custodial wallet supporting Solana or Base. Omnera does not custody funds at any point. The wallet establishes identity for the behavioral model without granting access to assets.

## First Session

The behavioral model begins building from the first interaction. It captures what the user views, how long attention holds on each opportunity, which positions are considered but not taken, and what is scrolled past. There is no configuration step.

What makes the first session significant is that it starts the compounding process. The model on session five is materially different from session one. The model on session thirty bears little resemblance to session five. Early access users begin accumulating behavioral signal before the general population has access.

## Scope

The feed, pre-trade intelligence, and cross-chain execution are available during early access. Feature availability may change prior to general release.
