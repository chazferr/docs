Every existing trading platform ranks assets by the same aggregate metrics: volume, price change, market cap. The result is an identical ranked list for every user. A professional trader managing six figures and a first-time buyer opening a wallet see the same screen. The platform has no mechanism to distinguish them.

This is not a data problem. The data exists. No platform captures it. They record transactions. They do not record decisions.

## What Omnera Observes

The behavioral model ingests session-level data that no transaction log contains.

- Scroll depth and velocity through different asset classes
- Dwell time on specific opportunities before any action is taken
- Near-executions that were initiated but not completed
- How often an opportunity is revisited across sessions before action occurs
- Instrument types the user gravitates toward: spot, perpetual futures, prediction markets
- The relationship between viewing behavior and execution decisions over time

A user who consistently views sub-$500K market cap tokens but only executes on those with locked liquidity is expressing a filter. A user who dwells on perpetual futures but exits before execution is expressing hesitation around leverage. A user who revisits the same prediction market three sessions in a row is expressing conviction building toward a threshold.

None of these signals appear in a transaction log. All of them appear in the behavioral model.

## How Relevance Is Ranked

Each opportunity in the feed receives a per-user relevance score derived from the behavioral model.

- The score determines rank. Two users opening the feed at the same moment see different content in a different order
- The model does not rely on stated preferences or manual configuration. It infers preferences from observed behavior
- Relevance scoring operates across all supported instrument types: spot tokens, perpetual futures, and prediction markets on Solana and Base
- The feed is a single interface. There is no switching between applications or chains to see the full opportunity set

## What the Feed Enables

- Differentiated discovery. The feed reflects individual behavior, not what is trending on aggregate
- All supported instruments in a single view across Solana and Base
- The interaction pattern is scroll, evaluate, execute. Discovery to trade occurs within a single screen
- Mobile-native design. The same behavioral model operates on mobile with the same ranking logic

## How Signal Compounds

Early sessions produce a coarse model. Signal density is low and relevance scoring is approximate. As session count increases, the model resolves finer distinctions between opportunity types, risk profiles, and market conditions.

- Behavioral data compounds. Each session adds signal that makes subsequent sessions more precise
- A trader on session fifty operates on a qualitatively different information surface than a trader on session three
- The model does not plateau at a fixed level of accuracy. New behavioral data continues refining relevance as market conditions and user behavior evolve

## Design Constraints

- The model begins learning from session one with zero configuration
- Behavioral inputs are captured regardless of whether a trade executes
- The feed covers spot, perpetual futures, and prediction markets in a single interface
- Wallet connection is non-custodial. Omnera never holds user funds
- No trade recommendations are generated. The model ranks. The user decides
